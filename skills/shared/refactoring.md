# Рефакторинг

## Уровни навыка
### Уровень 1
Рефакторинг носит стихийный характер, может выходить за предполагаемые границы.

Знаком с базовыми запахами кода вида большие методы и классы, длинный список аргументов функций и т.д..
Может применить на практике техники рефакторинга такого кода.

### Уровень 2
Соблюдает главное требование рефакторинга. Исходная функциональность не меняется при внесении изменений.

Руководствуется правилом бойскаута, для поддержания кода в чистоте.

Знаком с основными видами запахов кода. Способен решить большинство проблем, не требующих перепроектирования архитектуры или изменения модели данных.

### Уровень 3
Хорошо знаком с инструментами автоматизации для рефакторинга, предоставляемыми популярными IDE.

Необходимость рефакторинга оценивает комплексно. В оценке учитываются технические аспекты, значимость для бизнеса, интенсивность будущих доработок и связанные риски.

Рефакторинг, выполняемый в рамках бизнес-задач не приводит к существенному изменению сроков выполнения задачи.

Способен выполнить сложный рефакторинг, связанный с реорганизацией отдельного модуля приложения.
Способен выполнить декомпозицию планируемых работ.

### Уровень 4
Может целиком оценить картину проекта, выявив негативные паттерны, приводящие к накоплению технического долга.

Имеет опыт выполнения сложного рефакторинга, связанного с:

Перепроектирование архитектуры системы, изменение модели данных, публичного API.
Миграция системы на другой библиотеку, фреймворк и т.д.
Для сложного рефакторинга может составить план, оценить сроки и ресурсы, просчитать риски.
