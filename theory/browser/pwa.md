# Progressive Web Apps (PWA)

Если коротко — PWA это веб-сайт, который ведёт себя как нативное приложение. Его можно установить на рабочий стол или экран телефона, он работает офлайн, может присылать push-уведомления и быстро загружается даже на медленном соединении. При этом его не нужно скачивать из App Store или Google Play.

---

## Что такое PWA и зачем

**PWA (Progressive Web App)** — это веб-приложение, построенное на стандартных веб-технологиях, которое использует современные браузерные API для предоставления возможностей, ранее доступных только нативным приложениям.

Идея проста: разработчик пишет одну кодовую базу на HTML/CSS/JavaScript, а пользователь получает приложение, которое:
- Работает в браузере как обычный сайт
- Устанавливается на устройство и выглядит как нативное приложение
- Функционирует без интернета или при плохом соединении

### Зачем это нужно

Нативные приложения имеют преимущества: работают офлайн, быстро загружаются, интегрированы с ОС. Но у них высокий порог входа — пользователь должен найти приложение в магазине, скачать его (часто десятки мегабайт) и установить. Отток огромный: большинство пользователей уходит на каждом шаге.

Веб-приложения легко достичь через браузер или поиск, но они зависят от сети и не могут работать в фоне. PWA — попытка взять лучшее из обоих миров.

## Чем PWA отличается от обычного сайта и нативного приложения

| Характеристика | Обычный сайт | Нативное приложение | PWA |
|---------------|-------------|---------------------|-----|
| Установка | Не устанавливается | Из магазина приложений | С браузера, без магазина |
| Работа офлайн | Нет | Да | Да |
| Push-уведомления | Нет | Да | Да |
| Размер для скачивания | Нет | Десятки-сотни МБ | Минимальный |
| Одна кодовая база | Да | Нет (iOS + Android отдельно) | Да |
| Доступность через URL | Да | Нет | Да |
| Обновление | Мгновенно | Через магазин | Мгновенно |

Ключевое отличие от нативных приложений: PWA не нужно публиковать в магазин. Для пользователя «установка» означает добавить значок на экран — это делается прямо из браузера по предложению самого приложения.

## Ключевые характеристики PWA

### Устанавливаемость

PWA можно установить на устройство: появляется иконка на рабочем столе или экране телефона, приложение открывается в отдельном окне без адресной строки браузера — выглядит как нативное. На Android и desktop Chrome/Edge кнопка установки появляется автоматически, если приложение соответствует требованиям.

### Работа офлайн

Благодаря Service Worker PWA может кэшировать ресурсы и перехватывать сетевые запросы. Даже без интернета пользователь увидит что-то полезное — сохранённые данные, страницу «нет соединения» вместо пустого экрана браузера, или полную функциональность приложения в зависимости от реализации.

### Push-уведомления

PWA могут отправлять уведомления даже когда приложение закрыто. Для этого используются Push API и Notifications API в связке с Service Worker. Пользователь видит уведомления так же, как от нативных приложений.

## Базовые технологии

### Service Worker

Service Worker — это скрипт, работающий в фоне в отдельном потоке. Он не имеет доступа к DOM, но может перехватывать все сетевые запросы страницы, кэшировать ответы и обрабатывать push-уведомления.

Именно Service Worker обеспечивает работу офлайн: он перехватывает запрос, и если сеть недоступна — отдаёт закэшированный ответ.

```
Браузер → запрос → Service Worker → сеть (или кэш)
```

Service Worker регистрируется один раз, потом браузер запускает его при необходимости. Он живёт независимо от страницы.

### Web App Manifest

Manifest — это JSON-файл, который описывает приложение для браузера и ОС. Он говорит, как называется приложение, какая у него иконка, каким цветом красить строку статуса, в каком режиме запускаться.

```json
{
  "name": "Мой Заметочник",
  "short_name": "Заметки",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#ffffff",
  "theme_color": "#2196f3",
  "icons": [
    {
      "src": "/icons/icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    }
  ]
}
```

Manifest подключается в `<head>`:

```html
<link rel="manifest" href="/manifest.json">
```

Браузер читает manifest и решает, можно ли предложить пользователю «установить» приложение. Для этого должны выполняться базовые критерии: есть manifest с нужными полями, зарегистрирован Service Worker, сайт работает по HTTPS.

## Реальные примеры PWA

PWA используют крупные компании — и получают измеримые результаты:

- **Twitter Lite** — PWA-версия Twitter: в 1,5 раза больше страниц за сессию, в 3 раза меньше данных, +65% твитов. Размер приложения 600 КБ против 23 МБ нативного.
- **Starbucks** — PWA для заказа кофе работает офлайн: меню доступно без интернета, заказы синхронизируются при восстановлении соединения.
- **Nikkei** (японская газета) — +2,3× органического трафика, +58% подписчиков после перехода на PWA.
- **Hulu** — заменили десктопное Flash-приложение на PWA, получили +27% повторных посещений.
